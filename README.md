# drdroid.keyrights

Модуль управления защищёнными записями KeyRights для Bitrix24 On-Premise.

Текущий релиз: **2.1.5**.

Основной репозиторий: [github.com/elftorg/keyrights](https://github.com/elftorg/keyrights/).

## Возможности

- хранение логинов, паролей, URL и заметок в зашифрованном виде;
- разграничение доступа пользователей и групп Bitrix24;
- наследование прав разделов;
- журналирование операций и импорт/экспорт данных;
- AES-256-GCM с поддержкой чтения legacy-данных;
- API с централизованной авторизацией, CSRF-защитой и проверкой прав;
- шифрование промежуточного состояния импорта и строгая изоляция параметров
  GET/POST/JSON без чтения Cookie;
- копирование в буфер обмена через современный Clipboard API с безопасным
  fallback через `document.execCommand`, без Flash и ZeroClipboard;
- ORM-модели и индексы для служебных таблиц;
- локализация на русском, украинском и английском языках.

## Требования

- PHP 8.2 или новее;
- Bitrix24 On-Premise 20.5.400 или новее;
- расширение OpenSSL;
- MySQL/MariaDB или PostgreSQL, поддерживаемые вашей версией Bitrix24;
- Composer 2 для разработки и обновления зависимостей.

## Установка

1. Скачайте assets `drdroid.keyrights-<версия>.tar.gz` и `SHA256SUMS` из
   нужного [GitHub Release](https://github.com/elftorg/keyrights/releases).
   Не используйте архивы `Source code` — это архивы репозитория, а не пакет
   Bitrix-модуля.
2. Проверьте контрольную сумму в каталоге со скачанными файлами:

   ```bash
   sha256sum -c SHA256SUMS
   ```

3. Распакуйте пакет в отдельный каталог:

   ```bash
   mkdir keyrights-release
   tar -xzf drdroid.keyrights-<версия>.tar.gz -C keyrights-release/
   cp -a keyrights-release/drdroid.keyrights /bitrix/modules/
   ```

   В корне распакованного архива находятся инструкции и лицензия, а
   устанавливаемый модуль целиком расположен в папке `drdroid.keyrights/`.
   После копирования должен существовать файл
   `/bitrix/modules/drdroid.keyrights/install/index.php`.
4. В административной части Bitrix24 откройте Marketplace → Установленные
   решения (список модулей), найдите `drdroid.keyrights` и нажмите «Установить».
5. Сохраните автоматически созданную ключ-фразу либо задайте свою длиной не
   менее 16 символов, примите лицензию и подтвердите установку. Если поле
   оставить пустым, установщик безопасно создаст ключ-фразу на сервере.
6. Откройте `/keyrights/`.

Установщик зарегистрирует компонент `drdroid:keyrights` и скопирует его в
`/bitrix/components/drdroid/keyrights/`. Таблицы старого модуля не требуются:
чистая установка работает без них. Если обе таблицы `sib_kr_item` и
`sib_kr_right` существуют, установщик повторяемо переносит их данные в
`dr_kr_item` и `dr_kr_right`. Старые таблицы не изменяются и не удаляются.

Для новых установок серверный ключ рекомендуется передать через переменную
окружения `KEYRIGHTS_SERVER_KEY` (не менее 32 символов) до запуска установщика.
В этом случае ключ не сохраняется в таблице настроек Bitrix. Если переменная не
задана, установщик шифрует клиентский и серверный ключи штатным ключом
`crypto.crypto_key` из `.settings.php`. Только если штатная криптография Bitrix
не настроена, используется совместимое открытое хранение в настройках модуля.
После выбора источника ключа менять его без отдельной миграции нельзя:
существующие записи перестанут расшифровываться.

Установщик не обращается к внешним CDN и использует нативный JavaScript без
загрузки jQuery. До начала установки сервер проверяет СУБД и возможность
тестовой записи, версии PHP/Bitrix, криптографические функции, модули Bitrix и
исходные файлы. Отсутствующие целевые каталоги и файлы не требуются заранее:
возможность их создания проверяется временными файлами и каталогами в
существующем родителе. Существующие целевые файлы проверяются отдельно, а
каждая проблема выводится с точным путём. PHP cURL модулю не требуется.

## Разработка

```bash
composer install
composer test
```

Frontend находится в `install/components/drdroid/keyrights/`.

```bash
npm ci --prefix install/components/drdroid/keyrights
npm run build --prefix install/components/drdroid/keyrights
```

Подробности по архитектуре, безопасности и выпуску находятся в каталоге
[`docs/`](docs/).

## Релиз

Готовые архивы и их SHA-256 публикуются как assets соответствующего GitHub Release.
Каждый пакет содержит папку `drdroid.keyrights/` и лежащие рядом README,
инструкции, changelog и лицензию. В `/bitrix/modules/` копируется только папка
`drdroid.keyrights/`.
