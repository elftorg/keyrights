# Подтверждённые проблемы и краткий план исправлений

Дата проверки: 17 июля 2026 года. Проверены четыре отчёта из этого каталога,
код базового коммита `010bc8329a1e111f255f23e7f45ef08ff5b329f8` и рабочее дерево перед
выпуском 2.1.3. Повторяющиеся находки объединены.

## Не подтверждено как уязвимость

- GET для read-only маршрутов не является обходом CSRF: write-маршруты уже
  требуют POST, авторизацию и корректный `csrf_token`; запрет всего GET сломал бы
  штатную загрузку списков.
- `filemtime()` с путём от `DOCUMENT_ROOT` сам по себе не раскрывает этот путь
  клиенту. Подавление ошибок всё же удалено в пользу `is_file()`.
- Ограниченный двухмегабайтным входом разбор legacy-свойства не демонстрирует
  заявленного ReDoS.
- Отсутствие CSP на JSON API и совместимость с IE11 являются настройками
  усиления/совместимости, а не доказанными эксплуатируемыми уязвимостями.

Полноценный пентест работающего портала и миграции на реальных MySQL/PostgreSQL
остаются отдельными задачами; статические и контрактные тесты их не заменяют.
