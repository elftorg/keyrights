# Безопасность

- операции записи проходят через POST и CSRF-фильтр;
- API проверяет авторизацию и права владельца/администратора;
- обновление прав проверяет принадлежность записи целевому разделу;
- `rights/remove` не принимает произвольный объект без проверки доступа;
- favicon-запросы разрешают только публичные HTTP(S)-адреса и блокируют
  приватные, loopback и reserved-сети;
- новые шифротексты имеют версию и аутентифицированное шифрование
  AES-256-GCM;
- legacy-дешифратор сохраняется для чтения ранее созданных записей.
- копирование чувствительных значений выполняется только в браузере через
  Clipboard API или локальный `document.execCommand`; Flash, ZeroClipboard и
  отправка содержимого на внешний сервер не используются.
- отчёты об ошибках автоматически никуда не отправляются. Предложения и
  сообщения об ошибках публикуются вручную через
  [GitHub Issues](https://github.com/elftorg/keyrights/issues).

Секреты, ключевые фразы и production-конфигурация не должны попадать в
репозиторий, логи CI или релизные заметки.
